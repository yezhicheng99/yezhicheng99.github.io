hexo 博客的安装部署与多电脑同步管理

发表于2022-04-27|更新于2022-08-09|hexo
|阅读量:

Hexo 是一个非常方便好用的撰写博客的平台，但是有时候我们需要在不同电脑或者系统上撰写博客，这就会涉及到如何部署 hexo 在多台电脑上以及同步相关文档的问题。

在 A 电脑上部署 hexo

新建 hexo 文件夹并初始化博客

在 A 电脑的某个位置新建一个名为 blog 的文件夹，比如我的是 E:\blog

新建 github 仓库

配置 SSH Key，并将在本地生成的 ssh key 添加到 github

在 github 上新建一个仓库名为：xxx.github.io 的仓库，xxx 为你的 github 用户名，如我的是：yezhicheng99.github.io

新建分支用于保存 hexo 博客源文件

下载喜欢的主题并配置成 git submodule

这里我选的主题是：butterfly

手动打开 blog 文件夹下的 _config.yml 文件，找到 theme: 配置项并修改为

修改配置文件

手动打开 blog 文件夹下的 _config.yml 文件，修改一下内容

生成博客，并上传到 github 服务器

这时候，在浏览器中输入： yezhicheng99.github.io 就可以看到自己写的博客了 (这里可能会有延迟，如果没有更新一般等几分钟就可以了)

上传 hexo 源文件到 github 博客仓库下 hexo 分支

在 B 电脑上部署并同步 hexo 博客

下载 github 上 hexo 分支所有文件

安装必要项

如果报错没有关系，到这里你就可以书写新的博客了

书写新的博客并上传

打开 blog/source/_posts/ 目录下 “博客标题.md” 文件，并书写博客内容

上传 hexo 源文件到 github 博客仓库下 hexo 分支

注意：每次在写新的博客之前，一定要先在 blog 目录下执行 git pull　或者 git pull origin hexo:hexo，获取最新源文件
